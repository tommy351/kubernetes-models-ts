# @kubernetes-models/go-generate

## 0.1.0

### Minor Changes

- [#271](https://github.com/tommy351/kubernetes-models-ts/pull/271) [`13f4fa3`](https://github.com/tommy351/kubernetes-models-ts/commit/13f4fa3b1bd874e46a0ab2ec1a17170b736c5db8) Thanks [@tommy351](https://github.com/tommy351)! - Inline upstream struct types (`metav1.GroupKind`, `metav1.Timestamp`, `corev1.VolumeSource`, `admissionregistration/v1alpha1.Mutation`) whose JSON shape has no standalone TS module on the consumer side. Without this, `tsc --emitDeclarationOnly` fails with TS2307 when a third-party package references one of these types.

### Patch Changes

- [#271](https://github.com/tommy351/kubernetes-models-ts/pull/271) [`d30f6bb`](https://github.com/tommy351/kubernetes-models-ts/commit/d30f6bb379ebd538cf6a71d9db217a050300354a) Thanks [@tommy351](https://github.com/tommy351)! - Skip non-`$ref` entries in `allOf` instead of asserting. Inline `x-kubernetes-preserve-unknown-fields: true` entries (emitted by controller-tools for fields marked `+kubebuilder:validation:Schemaless` + `+kubebuilder:pruning:PreserveUnknownFields`) no longer crash the generator.

- [#271](https://github.com/tommy351/kubernetes-models-ts/pull/271) [`c6d948b`](https://github.com/tommy351/kubernetes-models-ts/commit/c6d948ba9c5310a69199e90eeb56dfadab2078a4) Thanks [@tommy351](https://github.com/tommy351)! - Break case-insensitive filesystem collisions. When sibling types in the same package emit to paths that fold to the same lowercase form (e.g. `TidbInitializer` and `TiDBInitializer` upstream), the root-kind variant keeps its natural file name and other variants get a stable `_<index>` suffix. Without this, `tsc --emitDeclarationOnly` fails with TS1149 on macOS/Windows filesystems.

- [#271](https://github.com/tommy351/kubernetes-models-ts/pull/271) [`0fe9a9f`](https://github.com/tommy351/kubernetes-models-ts/commit/0fe9a9f3021dfe6dface4bd33cf3ddcb6c7b136a) Thanks [@tommy351](https://github.com/tommy351)! - Extend `inlineIncludeStructs` with five more upstream types that have no standalone TS module on the consumer side: `metav1.GroupVersionKind`, `metav1.GroupVersionResource`, `admission/v1.AdmissionRequest` (webhook payload, never in the API server's OpenAPI spec), and `admissionregistration/v1alpha1.{ApplyConfiguration,JSONPatch}` (alpha group not republished by `kubernetes-models`). Without these, Kyverno's `AdmissionRequestInfoObject` and `MutatingPolicySpec` emitted imports against module paths that don't resolve.

- [#271](https://github.com/tommy351/kubernetes-models-ts/pull/271) [`c02bd6e`](https://github.com/tommy351/kubernetes-models-ts/commit/c02bd6e1d6a9d6f99a99286b9b8ad37e7bce9975) Thanks [@tommy351](https://github.com/tommy351)! - Qualify same-package refs in inlined struct schemas. Field refs to siblings of an inlined struct (e.g. `corev1.VolumeSource.awsElasticBlockStore → AWSElasticBlockStoreVolumeSource`) arrive from the controller-tools parser in unqualified form. Run `refPackageAdder` on the inlined schema with the source type's package as the fallback so `refNormalizer` can produce a proper `io.k8s.api.core.v1.*` ref instead of an empty-package one.

- [#271](https://github.com/tommy351/kubernetes-models-ts/pull/271) [`a97b359`](https://github.com/tommy351/kubernetes-models-ts/commit/a97b359ee92f74995ce8878f945617003c8a1a9b) Thanks [@tommy351](https://github.com/tommy351)! - When a definition file's path is renamed by the case-collision pass, also rename the emitted class and interface symbols (`ImageRef` → `ImageRef_1` etc.). Otherwise sibling renamed files in the same package all export the same `ImageRef` symbol, and the package-level `export * from` re-exports collide with TS2308.

- [#271](https://github.com/tommy351/kubernetes-models-ts/pull/271) [`dfb0cef`](https://github.com/tommy351/kubernetes-models-ts/commit/dfb0cef4cfff1b1e32a84c49e628e094b52cf6a9) Thanks [@tommy351](https://github.com/tommy351)! - Apply `pathRenames` to `_schemas/` paths too. Without this, types that survived a case-collision rename on the gen-tree side still collide at their schema-bundle path (`_schemas/<QualifiedClassName>.js`) because `getQualifiedClassName` produced an identical case-folded stem.

- [#271](https://github.com/tommy351/kubernetes-models-ts/pull/271) [`c3de28b`](https://github.com/tommy351/kubernetes-models-ts/commit/c3de28bf7c06ba6cf3155f0e9809a8e10acc68e3) Thanks [@tommy351](https://github.com/tommy351)! - Fall back to a plain type alias for non-GVK structs whose flattened body has a top-level `validate` property. Such a field would shadow the inherited `Model.validate()` method and fail TypeScript's `--emitDeclarationOnly` pass (e.g. Kyverno's nested `Rule` type, where `validate` carries the validation sub-spec).
